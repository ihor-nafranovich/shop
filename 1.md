ВВЕДЕНИЕ

Когда-то люди искренне не понимали, для чего может быть нужен сайт и зачем тратить на его создание немалые деньги. Это было на заре становления и популярности интернета. После сайты создавались не для очевидной выгоды, а скорее для развлечения. Однако нынешние времена четко определяют позицию сайтов, их важность для бизнеса и необходимость создания.
Главные преимущества создания сайта
Онлайн-офис 24 часа в сутки. Находясь постоянно онлайн, можно привлекать еще больше пользователей на свой сайт. Таким образом, сайт будет полезен и для вас, и для ваших клиентов, ведь на его страницы можно зайти в любое время, семь дней в неделю, круглый год.
Сайт – это лицо вашего бизнеса. Здесь можно предоставить всю необходимую информацию своим клиентам, оставить свои контактные данные для быстрой связи, поместить карту, на которой можно показать месторасположение офиса, указать часы работы, тарифы на услуги и многое другое.
Сайт может быть отличным рекламным инструментом для бизнеса. 25-30% клиентов узнают о вас посредством информации, размещенной на вашем сайте. Не стоит забывать о том, что с увеличением количества посетителей сайта это цифра будет увеличиваться.
Если сайт современный, на нем используются новейшие технологии, внедрены последние тренды, он может самостоятельно привлекать потенциальных клиентов.
В любом бизнесе у клиентов возникают идентичные вопросы. Не тратить свое время на ответы позволит сайт, благодаря которому есть возможность отвечать на самые популярные вопросы своих клиентов на отдельной странице. 
Справедливости ради следует отметить тот факт, что объявление в популярной рекламной газете увидят десятки тысяч людей. Сколько из них потенциальных клиентов – сказать трудно. На сайт же, как правило, попадут только целевые посетители, количество которых легко можно узнать из статистических данных ресурса. Таким образом, можно более гибко строить информационное наполнение ресурса. 
Цель дипломной работы – разработать и внедрить интернет-магазин техники Apple на основе фреймворка Laravel, что позволит увеличить продажи, отрегулировать объем онлайн‑заказов в зависимости от складских и логистических возможностей и запросов потребителей, улучшить степень осведомленности покупателей о реализуемой продукции, а также обеспечить возможность доступа к контактной и справочной информации всем заинтересованным лицам.
Для достижения вышеуказанной цели был поставлен ряд задач, которые предстоит решить, а именно:
Определение целей и задач интернет‑магазина техники Apple.
Анализ целевой аудитории и подробное исследование слабых и сильных сторон конкурентов.
Разработка концепции создания интернет-проекта, работа над его архитектурой.
Подготовка качественного графического и маркетингового материала.
Программная разработка с последующим тестированием адаптивной версии сайта на различных разрешениях.

1. АНАЛИТИЧЕСКИЙ ОБЗОР И ПОСТАНОВКА ЗАДАЧИ

Цели и задачи сайта

Веб сайт – бесплатный круглосуточный канал предоставления актуальной информации посетителям сайта, клиентам, партнерам и сотрудникам компании в неограниченном объеме и формате. Это сокращение расходов на другие виды рекламных носителей; неограниченные возможности демонстрации продукта; бесплатная и высокоинформативная обратная связь; возможность уточнения интересов потенциальных и постоянных клиентов; увеличение дохода компании за счет интернет-продаж; платформа для обучения и тестирования сотрудников; средство обмена конфиденциальной информацией между удаленными офисами. И еще много других преимуществ.
Своевременное обеспечение этих составляющих помогает компании зарабатывать позитивный имидж и доброе имя.
Обращаясь к разработчику, заказчики не могут объяснить, чего они хотят. Складывается впечатление, что они хотят всего и сразу. Ни один сайт не может решить все задачи, которые стоят перед компанией. Это просто невозможно.
Прежде, чем начать разработку интернет-страницы, нужно определить цель создания сайта. От нее будет зависеть формат, функционал и дизайн. Постановка цели – базовое условие при создании сайта для разработчика.
Представительство компании в интернете
Не все компании преследуют коммерческие цели. Потребность в создании сайта имеют государственные структуры, бюджетные и общественные организации, клубы по интересам и еще целый ряд компаний.
Цель создания сайта некоммерческих организаций – размещение основной информации, формирование интереса к себе. Возможно, презентация компании и ее услуг. В таком случае оптимальным форматом будет сайт-визитка или корпоративный сайт.
Интернет-продажи
Основная цель создания сайта для коммерческих структур – привлечение новых клиентов. Решить ее можно разными способами. Например, клиентов можно привлечь при помощи контекстной рекламы.
Сайт может быть нацелен на прямую продажу. Подойдет формат интернет-магазина или товарного каталога, официального сайта с кнопкой заказа. Современный и эффективный способ повышения интернет-продаж – создание лендинга.  
Брендинг
Часто целью создания сайта становится брендинг, т.е. формирование запоминающегося имиджа компании на рынке. Это одна из самых сложных задач, поэтому для ее решения требуется профессиональный мощный официальный сайт, разработанный с учетом корпоративного стиля.
Корпоративные сайты – дорогостоящее удовольствие. Оно подходит для состоявшихся компаний со стабильным доходом, имеющих готовую маркетинговую стратегию и план продвижения на рынке. Для молодых развивающихся компаний больше подходит формат сайта-визитки, на который потом можно «нарастить» дополнительные функции.
Бизнес-сайты для небольших компаний
Компании, работающие в оффлайне, часто создают сайты с целью информирования клиентов о новинках, акциях и бонусах. Для решения данной задачи лучше всего подходят бизнес-сайты. Они имеют все необходимые модули: от ленты новостей до подписки на рассылку. 
Другие цели создания сайта
У заказчика могут быть и другая цель создания сайта или целый комплекс задач. Современные технологии веб-разработки позволяют найти решение. Главное, при оформлении заказа донести до разработчика, какие именно задачи нужно решить? Для чего создается сайт? Только тогда можно надеяться на его эффективную работу.
Качественный интернет-проект способен решать самые серьезные задачи, а именно:
- увеличение лояльности клиентов;
- увеличение продаж за счет новых клиентов;
- сокращение расходов на службы технической и/или информационной поддержки;
- увеличение объема продаж;
- совершенствование и модернизация продукта и/или услуг;
- возможность получения обратной связи с клиентами, поставщиками, деловыми партнерами;
- рекламирование своих услуг посредством рекламных баннеров, маркетинговых текстов и т.д.
Основная цель создания интернет‑магазина техники Apple – комплексное решение всех поставленных задач. 
Сайт будет являться мощным инструментом онлайн‑продаж, который может значительно упростить взаимодействие с клиентами. Он позволит регулировать объем продаж в зависимости от складских остатков и логистических возможностей, благодаря сайту появится возможность автоматически принимать заказы и вовремя «поймать» желание клиента купить устройство Apple. Сайт даст возможность компании позиционировать себя как современного, технологичного и клиентоориентированного продавца.
Для достижения поставленных целей и задач потребуется:
- провести детальную проработку концепции;
- провести детальный анализ рынка, изучить конкурентов;
- четко разобраться с сильными и слабыми сторонами предлагаемого продукта;
- подробно заполнить бриф и создать прототип проекта;
- продумать навигацию и архитектуру сайта;
- разработать дизайн-макет проекта в соответствии с основными правилами;
- юзабилити;
- выполнить программную работу с использованием современных средств.

Анализ пользователей сайта


 Одной из основных задач маркетинга является анализ целевой аудитории. Целевая аудитория (target group, target audience) – это группа людей с определенными характеристиками, которые потенциально могут быть вашими клиентами или уже ими являются.
Если максимально верно выделить свою целевую аудиторию, можно направить рекламу на конкретных потребителей. Это приведет к успешному развитию бизнес-проекта, а также к увеличению процента продаж.
Ведь когда знаешь своего клиента, как говориться, в лицо, то значительно легче составить свою рекламную стратегию. А размытое представление о клиенте ведет к снижению эффективности торговли и снижению общей рентабельности проекта.
Кроме того, знание целевой аудитории позволяет снизить расходы на рекламу, сделав ее адресной.
Существует три главных группы интернет-посетителей веб-ресурса:
ориентированные на получение необходимой информации;
выбирающие подходящий товар или же услугу;
ориентированные на покупку.
Помимо целевой, у веб-ресурса имеется и косвенная аудитория. Это те клиенты, которые попадают на ресурс, введя в строку поисковика смежный по смыслу запрос.
Они могут превратиться в потребителей компании, но это происходит достаточно редко.
Чтобы определить именно вашу целевую аудиторию необходимо правильно составить портрет целевого посетителя. Для этого стоит ответить на следующие вопросы:
Пол посетителя, его возраст, его уровень материального дохода, семейное положение, род занятий.
В каких социальных сетях зарегистрирован посетитель, на каких форумах?
Какие проблемы посетителя сможет решить ваша услуга или товар?
Какие эмоции вызовет?
Причины купить именно у вашей компании.
Сегментировать, другими словами, выделить целевую аудиторию необходимо для ее определения.
Сегментация (кластеризация) – сортировка потенциальных клиентов на группы с похожими характеристиками, раскрытие потребностей группы и формирование того предложения, которое будет ориентировано на каждую группу.
Результаты кластеризации позволят выявить целенаправленный рыночный кластер. А потребители, которые в него входят, и будут являться целевой аудиторией.
Кластеризация  позволяет персонализировано использовать рекламные возможности, исходя из запросов потенциальных клиентов.
Привлечение посетителей на страницу совершается с помощью продвижения ресурса в интернете. Ведь имея на руках составленный портрет своего клиента значительно легче продвинуть сайт.
Целевую аудиторию можно выделить на основе нескольких характеристик:
географические: регион проживания, город, район, плотность населения, климат и т.д. Географическое сегментирование – наиболее простой метод сегментирования, но достаточно важный.
социально-демографические: пол, возраст, уровень дохода, образование, социальный статус, размер семьи, количество детей и их возраст и т.д.
психографические: стиль жизни, особенности личности, черты характера, жизненная позиция, доминирующие мотивы поведения, образ жизни, система ценностей. Психография классифицирует людей по отношению к жизни и их покупательским привычкам.
поведенческие: повод для совершения покупки, отношение к товару, доминирующие мотивы покупки, приверженность к определенным маркам, частота пользования конкретным товаром, способность откликнуться на новые товары на рынке.
Информация о целевой аудитории обычно собирается через маркетинговые исследования с использованием таких инструментов, как наблюдение, опрос и анкетирование.  
При составлении примерного портрета целевого посетителя интернет‑магазина техники Apple воспользуемся следующими характеристиками:
лица женского и мужского пола, имеющие возраст от 16 до 60 лет. При этом акцент следует сделать на студентах и молодых специалистах (возраст до 30 лет), а также на специалистах в сфере IT и креативных индустрий 25-45 лет, которые, согласно проведенному исследованию, являются наиболее активными покупателями техники Apple, со средним и высоким уровнем дохода;
мужчины и женщины, проживающие в г. Минске и крупных городах Республики Беларусь, а также иностранные гости столицы;
уровень дохода – средний/выше среднего.

Анализ аналогичных сайтов

На начальном этапе создания сайта помимо определения целевой аудитории сайта немаловажным является анализ сайтов конкурентов.
Изучая сайты конкурентов, можно увидеть их преимущества по сравнению с собственным сайтом, следовательно, получить полезную информацию о том, что нужно изменить у себя на сайте.
Анализ сайтов конкурентов можно условно разделить не несколько этапов: 
Анализ внутренней среды сайта:
– технический анализ сайта (доменное имя, кода сайта, скорость  
– маркетинговый и seo анализ сайта (контент сайта, ключевые слова на сайте, дизайн сайта, оптимизация сайта, структуры сайта).
Анализ внешней среды сайта:
– поисковый анализ сайта (анализ позиций сайта в поисковиках, анализ внешних ссылок на сайт).

1.4. Бриф сайта и техническое задание

Составление брифа сайта – важнейший пласт работы, к которому нужно отнестись очень ответственно, чтобы избежать больших неприятностей в будущем. В брифе необходимо максимально развернуто ответить на представленные вопросы, которые помогут дизайнеру и разработчику лучше понять ваш замысел.
Порой может показаться, что в брифе скучные и глупые вопросы, однако это не совсем так. Правильно составленный бриф должен отличаться простотой и лаконичностью, и затрагивать все возможные проблемные моменты. Не бывает так, чтобы цветовая гамма определялась исходя из соображений дизайнера, а вопрос расположения блоков и размер футера оставался на усмотрение разработчика. Заказчик должен обозначить свое видение и восприятие будущего сайта, а подрядчики в случае чего должны его скорректировать и вместе выработать идеальную концепцию разработки ресурса.

Техническое задание

Общие положения
Предметом разработки является интернет‑магазин техники Apple «iShop». Сайт должен выполняться на русском языке.
Назначение сайта – представление информации о продаваемой технике Apple и аксессуарах, условиях покупки, доставки и сервисного обслуживания, контактной информации, а также сопутствующей информации по теме проекта в интернете.
Цель создания сайта – создание онлайн‑площадки для продажи техники Apple, привлечения потенциальных клиентов, увеличения объема онлайн‑продаж и обеспечения возможности доступа к информации о товарах, акциях, условиях доставки, а также контактной и справочной информации всем заинтересованным лицам.
Реализуемый функционал сайта:
редактирование и развитие информационных ресурсов с помощью системы удаленного администрирования сайта посредством сети Интернет (административная панель Laravel E‑Shop);
создание, размещение, структурирование и хранение в единой базе данных (MySQL) информационных ресурсов Заказчика с использованием ORM Eloquent;
интерактивная работа с пользователями за счет применения Livewire‑компонентов (каталог, корзина, поиск, личный кабинет).
В результате должен быть создан информационный ресурс с современным адаптивным дизайном, построенный на стеке PHP 8.2, Laravel 12, Livewire 3 и Bootstrap 5, а также полным набором функциональных характеристик и сервисов для удобной навигации на сайте. Доступ к ресурсам сайта должен быть представлен пользовательской и административной частями, что позволит эффективно и безопасно наладить работу с пользователями. 
Основные функции
Пользовательская часть сайта будет включать:
 систему навигации по сайту с возможностями передвижения по структуре сайта с любой страницы;
 поиск по сайту с любой страницы;
 регистрацию на сайте для возможности оставить отзыв или сделать предзаказ;
 консультации на сайте;
 обучающие мастер-классы.
Требования к содержанию разделов сайта:
Главная страница должна содержать следующие блоки:
Хедер – это верхний блок любой веб–страницы, который называется также шапкой сайта и находится противоположно футеру (подвалу сайта). Обычно в нем располагается название бренда, логотип, панель навигации и базовые контакты. Хедер выступает одним из ключевых элементов оформления сайта. В данном блоке находится слайдер.
Блок отображения главного меню сайта. Меню должно обеспечивать доступ ко всем разделам сайта. Меню должно поддерживать иерархическую структуру, дочерние элементы меню должны быть “выпадающими”. 
Футер сайта – это область, расположенная внизу каждой страницы сайта под основным контентом, место, куда пользователи смотрят при поиске конкретных элементов. Футер – это ограниченное количество пространства экрана, которое должно быть посвящено только важной информации. В данном блоке необходимо разместить информацию о разработчике и иконки социальных сетей, чтобы можно было поделиться информацией.
Второстепенные страницы сайта являются типовыми и должны содержать следующие блоки:
Хедер (как и на главной странице).
Блок отображения контента по теме страницы.
Футер сайта (как и на главной странице).

Каждая страница сайта должна содержать название организации.

Требования к графическому дизайну
Сайт должен иметь оригинальный дизайн, не препятствующий быстрой загрузке его страниц. 
Дизайн должен быть выдержан в светлых тонах. Использовать преимущественно белый, светло‑серый и темно‑серый цвета и их оттенки, также должны присутствовать аккуратные акцентные цвета (синий, зеленый) для выделения важных элементов интерфейса.
Основной визуал сайта должен соответствовать тематике деятельности (продажа техники Apple) и иметь с ней непосредственные ассоциации.
Специальных требований шрифтовому оформлению нет.
Ключевые слова, характеризующие дизайн сайта: уютный, стильный, запоминающийся, профессиональный.
На первой странице не должно быть большого объема текстовой информации.
Требования к навигации
Для навигации должно использоваться главное меню, а также дополнительное меню главной страницы.
Главное меню представляет собой панель из ссылок (кнопок) с раскрывающимися при наведении курсора мыши выпадающими меню и эффектом визуального выделения пункта меню (эффект rollover). 
Галерея главного меню и все дочерние пункты в выпадающих меню, раскрывающихся при наведении мыши, являются ссылками, служащими для перехода на соответствующую страницу.
Переход на главную страницу сайта осуществляется с каждой страницы сайта.
Технологические требования
Сайт разрабатывается как веб‑приложение на PHP 8.2 с использованием фреймворка Laravel 12, должен корректно отображаться в современных браузерах и адаптироваться под различные разрешения экранов (desktop, tablet, mobile).
Мета-теги и контент сайта на этапе изготовления сайта должны быть настроены для поисковых систем, чтобы обеспечить продвижение сайта по ключевым словам в поисковых системах Yandex и Google.
Для управления наполнением сайта используется встроенная система администрирования, реализованная на Laravel (маршруты `/admin/*`, Livewire‑компоненты и Blade‑шаблоны), которая представляет собой интерфейс, позволяющий редактировать все разделы и подразделы сайта. 
Для реализации пользовательского интерфейса используются Blade‑шаблоны Laravel, язык разметки HTML5 и каскадные таблицы стилей (CSS3/Bootstrap 5). Исходный код должен разрабатываться в соответствии со стандартами W3C.


Требования к верстке
Html-документ должен соответствовать стандарту W3C (HTML 5.0), и быть сверстан с применением CSS.
Html-код сайта должен быть удобен для понимания и структурирован, сложные и неоднозначные моменты прокомментированы. 
Страница должна максимально идентично отображается во всех современных браузерах: Internet Explorer 8.0 и выше, Mozila FireFox 2.0 и выше, Opera 8.0 и выше, Google Chrome и при разрешениях монитора от 1024x768 до 1600х1200. 
Все стили следует вынести в файл styles.css, определение стилей непосредственно на странице недопустимо.
Все java-скрипты следует хранить в папке /js/, вставка скриптов непосредственно в html-код недопустима, за исключением ситуаций, когда вынос скриптов в отдельный файл невозможен или нерационален.
Все названия стилей должны быть английскими (без русских слов на латинице).
Все тэги должны быть написаны в нижнем регистре.
У всех ссылок должен быть прописан параметр title="".
У всех картинок должен быть прописан параметр alt="". 
Не следует использовать на странице заголовки h2 если нет заголовка h1 (это касается всех уровней заголовков).
Не использовать на странице более одного заголовка h1.

Информационная структура

Информационная структура имеет немаловажное значение при разработке сайта. Если ее не продумать, появляются «лишние» страницы, дублирующие ссылки на сайте и прочие малоприятные нюансы. Поэтому разработка информационной структуры сайта требует должного внимания.
Что скрывается под понятием «информационная структура сайта»? В общем понимании – это способ организации информационных материалов сайта вместе со всеми связями, благодаря которым эти материалы взаимодействуют друг с другом. В зависимости от того, как организовано данное взаимодействие, будет обеспечено комфортное пребывание посетителей при использовании сайта.
Главное, что надо учесть при разработке информационной структуры сайта – это его модель. По числу информационных материалов для размещения на сайте можно судить о том, линейная или иерархическая, а может быть и запутанная («паутина») модель будет выбрана. Ведь каждая из них имеет свои преимущества. 
Линейная структура – самая простая из всех. Веб-страницы идут одна за другой, и пользователь просматривает их как слайд-шоу. Здесь нет разделения информационного содержания на уровни. Все страницы равноправны, и каждый посетитель их должен увидеть. Реализация проста, но минусов много. Потому она имеет четкие границы применения: имиджевые сайты-презентации и онлайновые учебные пособия.
Простейший вариант сайта – набор HTML-страниц, каждая из которых имеет ссылку на последующую и предыдущую, кроме крайних.
Существует линейная структура с возможностью выбора вариантов. Основное отличие от линейной структуры – выбор между двумя ветками. Чаще всего это используют для сбора информации о посетителе. 
Линейная структура с альтернативами и вариантами удобна в случае необходимости контроля деятельности посетителей. И у посетителей есть возможность проявить инициативу, что уже является некой свободой выбора и облегчает доступ к необходимой именно им информации.
Иерархическая структура сайта, больше известная как «дерево» – это универсальная модель организации сайтов, подходит для многих типов сайтов, в том числе таких, как веб-представительства. Данная организация сайта дает посетителям возможность управления глубиной посещения сайта.
Ее принцип прост: пользователь при попадании на главную страницу предстает перед выбором, куда двигаться дальше. Переходя в нужный раздел, он выбирает нужный подраздел и так далее. У древовидной структуры очень много плюсов, но есть и существенный минус: достаточно сложно соблюдать баланс между глубиной и шириной. 
В классическом «дереве» один информационный материал принадлежит лишь одной подкатегории, категории или разделу.
Решетчатая структура. Она на порядок сложнее всех вышеописанных. Здесь страницы размещены на различных ветках. Но у пользователя есть возможность перемещения по ним как вертикально (вверх-вниз), так и горизонтально (между ветками на разных уровнях). Используют решетку в основном только в каталогах. Использование такой структуры в других проектах нецелесообразно. 
Какие же разделы и категории должна содержать информационная структура сайта для бизнеса? Единого мнения в этом вопросе нет, ведь каждая компания или организация индивидуальна. Однако информационная структура сайтов зачастую включает в себя «стандартные» разделы и категории без учета специфики этих сайтов. 
Разработка грамотной информационной структуры лежит в основе успеха и популярности информационного ресурса. Данный этап планирования требует комплексной и серьезной работы многих специалистов, поскольку на нем и будет в дальнейшем основываться дизайн сайта, его продвижение, развитие и поддержка.

2.2 Проектирование макета сайта 

Прототип – это схематичный макет (набросок, эскиз) страницы или страниц сайта, с изображенными на нем всеми элементами дизайна (кнопками, изображениями, меню, формами и т.д.). Бывает двух видов статичный (изображение) и динамичный (Html).
Как известно многие ошибки которые возникают на начальном этапе разработки того или иного проекта, связанны с неполным пониманием проекта. Упускаются многие детали из внимания, которые в последствие могут дорогого стоить. 
Для того чтобы избежать или минимизировать эти ошибки и производится прототипирование.  Прототипирование сайтов дает нам возможность быстро построить четкую структуру, проанализировать ее. Выявить все недочеты и в кротчайшие сроки устранить, с минимальными потерями времени и финансовых средств.
Для этого производится анализ контента, из этого анализа получаем структуру, чтобы четко представить структуру и понять все ли верно мы делаем, в верном ли направление идем и помогает прототипирование.
На этапе прототипирования сайта детализация интерфейса и остальных мелочей не очень важны, важно построить правильную структуру и понять, как она будет взаимодействовать с пользователями.
Также не стоит забывать о том, что дизайнеру будет гораздо проще разработать дизайн сайта по прототипу нежели по брифу, ТЗ.
К прототипам обычно применяют такие требования как: 
высокая скорость создания прототипа. Быстрое прототипирование позволяет сэкономить время на разработку проекта.
детализация (не для всех видов прототипов).
легкость и скорость внесения изменений в прототип.
интерактивность (возможность прототипа реагировать на действие пользователя).
Доступность для всех участников процесса (заказчик, дизайнер, менеджер, программист и т.д.).
Методы прототипирования сайтов
Существует множество различных методов прототипирования сайтов, в частности:
бумажное прототипирование;
прототипирование при помощи специальных программ (Axure Pro, WireframeSketcher, SketchFlow и т.д.);
прототипирование при помощи графических программ (Adobe Photoshop, Adobe Illustrator, InDesign и т.д.).
Бумажное прототипирование – один из самых простых и быстрых методов создания прототипов сайтов, со своими плюсами и минусами. Для того чтобы создать прототип достаточно взять лист бумаги и ручку (можно несколько цветов для наглядности) или карандаш.
Из плюсов данного метода можно отметить:
высокую скорость создания прототипа;
возможность создавать довольно детализированные прототипы;
не требуются специальные знания (программирование, владение графическими редакторами);
оптимальная скорость изменения прототипа, за счет перерисовки;
возможность оставлять комментарии в любых местах прототипа любым из участников разработки проекта.
Минусами данного метода являются:
отсутствует интерактивность у прототипа, может не выявить ряд проблем в интерфейсе;
внесение изменений не всегда возможно, приходится перерисовывать;
недостоверность размеров элементов на прототипе, возможны последствия при создании дизайна сайта;
не всегда имеет эстетичный вид, за счет чего может не понравиться заказчику.
Прототипирование при помощи специальных программ (Axure Pro, WireframeSketcher, SketchFlow и т.д.). Axure Pro – достаточно простая программа, в которой страницы организованны в древовидной форме. Очень гибкая и удобная в обращение программа. Скорость работы значительно увеличена за счет возможности повторного использования объектов (формы, картинки, текст и т.д.). Каждый объект имеет свои свойства, которые можно изменять. Прототипы получаются эстетичными и интерактивными. Скорость изменения достаточно высокая, за счет возможности создания своих объектов конкретных областей будь то меню, хедер, футер, которые с легкостью можно повторно использовать. Возможность частичного юзабилити тестирования.
Из плюсов данного метода можно отметить:
высокую скорость создания прототипа;
высокая детализация прототипа;
прототипы имеют эстетичный вид;
прототипы со средней интерактивностью;
высокая скорость внесения изменений без повторной отрисовки прототипа;
возможность создания прототипа в виде Html или картинки;
полная доступность для всех участников разработки проекта.
Минусы данного метода:
минусов не обнаружено, разве что изучение программы.
Прототипирование при помощи графических программ (Adobe Photoshop, Adobe Illustrator, InDesign и т.д.).
Из плюсов данного метода можно отметить:
средняя скорость создания прототипа;
высокая детализация прототипа;
прототипы имеют эстетичный вид;
высокая скорость внесения изменений без повторной отрисовки прототипа;
полная доступность для всех участников разработки проекта.
Минусы данного метода:
отсутствует интерактивность у прототипа, созданного в Adobe Photoshop, может не выявить ряд проблем в интерфейсе.  В некоторых программах, таких как InDesign возможно создать низкую интерактивность;
требуются знания по пользованию графическим редактором.
При разработке прототипов необходимо понимать, что не каждый метод будет оптимален в той или иной ситуации, и поставленной задаче. Лучше учитывать все параметры при выборе метода прототипирования сайтов. И тогда создаваемые проекты будут отличаться продуманностью, хорошим юзабилити, а также будут удобны для целевой аудитории.

2.3 Проектирование графического дизайна сайта

Дизайн сайта – это оформление контента, совокупность всех графических элементов на веб-странице. Раньше под веб-дизайном понимали исключительно визуальное оформление, но теперь на первый план вышло удобство пользователя, поэтому к задачам веб-дизайнера прибавились аналитика и грамотное структурирование информации на сайте.
Основная задача дизайна – познакомить пользователя со страницей. Он облегчает взаимодействие пользователя с веб-страницей, а, значит, положительно влияет на конверсию и поведенческие факторы. Продуманный дизайн создает нужное впечатление о компании.
Клиент, попавший на страницу, должен легко и быстро находить нужную информацию, в противном случае он просто уйдет к конкурентам. Поэтому проектирование графического дизайна сайта – один из важнейших этапов в становлении готового проекта, ведь прежде чем по достоинству оценить все плюсы грамотной реализации технических аспектов, каждый пользователь, заходя на страницу, оценивает визуальную часть проекта, логику расположения его ключевых элементов, удобство чтения, просмотра графики, прокрутки и так далее. 
Грамотное расположение графических элементов и текстовых блоков, продуманный выбор цветов и форм, привлекательный внешний облик сайта, соответствующий его тематике, – все вместе это гарантия положительного интереса со стороны пользователей сети Интернет.
Концепция графического дизайна отражает особенности тематики проектируемого сайта. Ключевые слова, характеризующие дизайн сайта, – современный, технологичный, запоминающийся, удобный, информативный.
Цветовая гамма сайта подобрана таким образом, чтобы подчеркнуть надежность, премиальность, технологичность и, в то же время, побудить пользователя к действию, привлечь внимание.
Для этого используются следующие цвета:
Белый цвет – основной фон, ассоциируется с чистотой и минимализмом.
Черный – используется для акцентных элементов и текста, создаёт ощущение строгости и элегантности.
Серый цвет и его оттенки – нейтральный фон для блоков и карточек товаров, не отвлекает от основного контента.
Акцентные цвета (синий, зелёный) – применяются для кнопок и интерактивных элементов («Купить», «Оформить заказ»), привлекая внимание к целевым действиям.

2.4 Юзабилити сайта

Юзабилити – степень удобства использования продукта – является одним из ключевых показателей для оценки любого digital продукта. Иными словами, юзабилити – это эргономика в цифровой среде. Хорошее юзабилити делает опыт пользователя сайта понятным, результативным и непрерывным.
Интерфейс с высоким уровнем юзабилити отличается:
Изучаемостью. Благодаря логично расположенным элементам и знакомым паттернам взаимодействия с ними, пользователь быстро осваивает интерфейс сайта, не прилагая для этого существенных усилий.
Эффективностью. Любой пользователь, посетивший сайт, стремится выполнить определенную задачу. Если ему удалось достичь цели быстро и легко, навигация сайта и организация контента отвечает требованиям юзабилити.
Минимальным количеством ошибок. Любая ошибка нарушает непрерывность опыта пользователя и вынуждает его повторять выполненные действия снова. Задача UX-оптимизатора – свести к минимуму вероятность возникновения таких ошибок, а также предусмотреть их простое исправление – например, в случае с некорректно заполненной формой.
Эстетичным дизайном. Критерий эстетики субъективен, однако внешний вид сайта должен соответствовать сложившимся нишевым стандартам. Новаторство никто не отменял, но здравый смысл должен быть на первом месте.
Современные пользователи потребляют информацию с невероятной скоростью – всего за несколько секунд человек может решить, стоит ли ему оставаться на сайте или нет. Как бы ни был дорог сайт вам самим, для обычного пользователя это просто еще одна страница из нескольких десятков уже просмотренных за сегодня. Он не станет внимательно его изучать, пытаясь оценить достоинства интерфейса или контента – вместо этого он за несколько секунд сделает выводы относительно того, может ли сайт помочь ему выполнить определенную задачу.
Попадая на страницу, пользователь быстро «сканирует» ее, обращая внимание на самые привлекательные элементы, фразы и изображения. По мере движения взгляда посетителя от шапки страницы к футеру, ресурс его внимания стремительно заканчивается.
При этом большинство пользователей незамедлительно кликают по элементу, который показался им интересным или наиболее подходящим. Пользователи делают это, не вдаваясь в подробности и не анализируя все доступные на вашем сайте опции.
Только понимая особенности поведения и психологии пользователей, можно создать продукт с качественным юзабилити.
Разработка пользовательского интерфейса сайта производилась с учетом всех основных требований юзабилити, так как выбор сайта пользователем производится не только по его информативности, но и по тому, насколько удобно им пользоваться.
При разработке учтены следующие основные требования юзабилити:
 Оформление:
не применяется на сайте графика, которая не несёт пользы, а наоборот усложняющая восприятие;
яркими цветами только расставлены акценты в небольших областях; 
для фона – нейтральные цвета (белый и серый в моём случае);
 Текст:
текстов на темном/ярком фоне, длинных текстов с низким контрастом цвета шрифта и цвета фона нет; 
грамматических ошибок нет; 
выделенные заголовки;
небольшие абзацы, для лёгкого прочтения.
Взаимодействие:	
сайт показывает возможность воспринять действия пользователя (при подведении курсора ссылки и кнопки изменяют вид);
на все значимые графические изображения установлен соответствующий альтернативный текст; 
информация организована так, чтобы наиболее важные данные находились в начале страницы; 
ни один элемент не называется по-разному в разных местах.
Навигация:
навигация доступна на каждой странице;
лёгкость перемещение по главной странице (боковой лэндинг);
все страницы имеют корректный заголовок (title).

3.1 Выбор технологий для разработки сайта

Любой WWW-документ может содержать стилизованный и форматированный текст, графику и гиперсвязи с различными ресурсами Интернет. Чтобы реализовать все эти возможности, был разработан специальный язык описания WWW-документов, названный HyperText Markup Language (HTML), то есть, «язык разметки гипертекста».
Описание веб-страницы на языке HTML представляет собой набор инструкций, интерпретируемый программой-браузером.
Документ, написанный на HTML, представляет собой текстовый файл, содержащий собственно текст, несущий информацию читателю, и теги разметки (markup tags). Теги – это определенные стандартом HTML, последовательности символов, являющиеся инструкциями для  браузера.
Согласно этим инструкциям программа располагает текст на экране, включает в него рисунки, хранящиеся в отдельных графических файлах, и формирует гиперсвязи с другими документами или ресурсами Интернет. Таким образом, файл на языке HTML приобретает облик Web-документа только тогда, когда он интерпретируется браузером.
Для создания сайта использовались следующие веб-технологии:
– HTML5
HTML5 стал необходимой, вынужденной эволюцией языка, на котором пишутся веб-страницы. Он был разработан для написания веб-приложений – динамических интерактивных веб-страниц. Его предшественник, HTML4, появившийся в конце 90х, предназначался для веб-страниц (статических документов, связанных между собой гиперссылками и содержащих текст, изображения, формы и т.д. ).
Ключевые понятия:
Его разработали для обеспечения взаимодействия страниц в разных браузерах. Сегодня мы используем различные браузеры (к примеру, на работе у нас может стоят IE, в телефоне – Safari или Opera, а дома –Firefox) и было бы неразумно и досадно, если бы сайт не работал бы в чем-то из них.
Все производители браузеров  – Opera, Mozilla (Firefox), Apple (Safari), Microsoft (Internet Explorer), Google (Chrome) – все они сотрудничают между собой, равно как и с еще массой организаций и частных лиц: с Netflix, Adobe, IBM, HP, BBC и многим другими.
Он разработан, чтобы расширить возможности современного веба, не нарушая при этом работы уже существующих веб-страниц.
Он может соревноваться с плагинами типа Microsoft Silverlight и Adobe Flash, которые сами по себе были созданы, чтобы закрыть дыры в окаменелом стандарте HTML4.
В работе используется 5 версия языка разметки.
А именно использование новых семантических тегов, позволяющих более осмысленно организовывать внутреннюю структуру веб-страниц. Это включает как блочные теги вроде header, footer, article, так и теги для разметки текста, например, mark, ruby, details. 
HTML в чистом виде имеет весьма ограниченный набор инструментов, не позволяющий решать те или иные дизайнерские и функциональные задумки. Для стилевых описаний, тех или иных HTML тегов используется CSS (Cascading Style Sheets) – каскадные таблицы стилей, который может быть применён как к отдельному тегу – элементу, так и одновременно ко всем идентичным элементам на всех страницах сайта [8]. CSS – своего рода дополнение к HTML, которое значительно расширяет его возможности.
CSS это язык стилей, определяющий отображение HTML-документов. Например, CSS работает с шрифтами, цветом, полями, строками, высотой, шириной, фоновыми изображениями, позиционированием элементов и многими другими вещами. CSS, на сегодняшний день, поддерживается всеми браузерами (программами просмотра).
Преимущества CSS:
позволяет значительно сократить размер кода и сделать его читабельным;
позволяет задавать такие параметры, которые нельзя задать только языком HTML. Например, убрать подчеркивание у ссылок;
позволяет легко изменять внешний вид страниц;
с CSS связана так называемая блочная верстка сайта.
– CSS 3
CSS3 – это новый стандарт оформления HTML документов значительно расширяющий возможности предыдущего стандарта CSS2.1.
Многие возможности, которые были труднодоступны в CSS2.1, то есть требовали использования дополнительных внешних программ (таких как Adobe Photoshop), скриптов (таких как JavaScript) или специальных "хитростей" могут легко достигаться в CSS3 за счет использования новых свойств оформления.
В CSS3 Вы можете:
создавать элементы со сглаженными углами;
создавать линейные и сферические градиенты;
более гибко оформлять фоновую картинку элементов;
добавлять к элементам и к тексту элементов тени;
использовать небезопасные шрифты (не боясь при этом, что они будут не поддерживаться браузером пользователя);
создавать анимацию и различные эффекты переходов;
задавать цвета несколькими новыми способами и многое другое.
–  Bootstrap 
Bootstrap – это открытый HTML, CSS и JS фреймворк, который используется веб-разработчиками для быстрой верстки адаптивных дизайнов сайтов и веб-приложений. Его преимущества:
скорость работы – благодаря множеству готовых элементов верстка занимает значительно меньше времени;
масштабируемость – добавление новых элементов не нарушает общую структуру;
легкая настраиваемость – редактирование стилей производится путем создания новых css-правил;
большое количество шаблонов;
огромное сообщество разработчиков;
широкая сфера применения – Bootstrap используется в создании тем для практически любой CMS.
– Javascript
JavaScript – это созданный фирмой Netscape межплатформенный объектно-ориентированный язык скриптинга (сценариев). JavaScript это небольшой компактный язык; он не используется для создания самостоятельных приложений, а разработан специально для упрощенного внедрения в другие приложения и продукты, такие как веб-браузеры. Находясь внутри рабочей среды-хозяина, JavaScript может подключаться к объектам этой среды для предоставления программного управления ими.
– PHP 8.2, Laravel 12, Livewire 3 и MySQL
PHP 8.2 – современная версия скриптового языка программирования общего назначения, интенсивно применяемая для разработки веб‑приложений. Является одним из лидеров среди языков программирования, используемых для создания динамических веб‑сайтов.
Laravel 12 – популярный PHP‑фреймворк, предоставляющий MVC‑архитектуру, маршрутизацию, систему миграций, ORM Eloquent, механизм очередей и событий, что существенно ускоряет разработку и упрощает сопровождение проекта.
Livewire 3 – библиотека для Laravel, позволяющая создавать динамические интерфейсы без написания JavaScript, реализует взаимодействие компонентов (каталог товаров, фильтры, корзина, поиск, личный кабинет) с сервером через AJAX‑запросы.
MySQL – реляционная система управления базами данных, используемая для хранения информации о пользователях, товарах, заказах, категориях и других сущностях интернет‑магазина. Обычно используется в качестве сервера, к которому обращаются локальные или удалённые клиенты.

4. ТЕСТИРОВАНИЕ САЙТА

Тестирование интернет‑магазина техники Apple проводилось с целью проверить соответствие реализованного Laravel‑приложения требованиям технического задания и ожиданиям пользователей, а также оценить устойчивость работы системы в разных окружениях.

4.1 Функциональное тестирование

Функциональное тестирование охватывало основные пользовательские и административные сценарии:
- проверка работы маршрутов и Livewire‑компонентов пользовательской части:
  - главная страница (`/`, компонент `App\Livewire\HomeComponent`);
  - каталог по категориям (`/category/{slug}`, компонент `App\Livewire\Product\CategoryComponent`);
  - карточка товара (`/product/{slug}`, компонент `App\Livewire\Product\ProductComponent`);
  - корзина (`/cart`, компонент `App\Livewire\Cart\Cart`, иконка корзины `CartIconComponent`, модальное окно `CartModalComponent`);
  - оформление заказа (`/checkout`, компонент `App\Livewire\Cart\CheckoutComponent`);
  - регистрация (`/register`) и авторизация (`/login`), личный кабинет (`/account`), список заказов (`/orders`) и просмотр заказа (`/order-show/{id}`) через компоненты пространства имён `App\Livewire\User`;
- проверка корректности создания, изменения и удаления записей в базе данных через модели `Product`, `Category`, `Order`, `OrderProduct`, `User`, `Filter`, `FilterGroup`;
- проверка формирования заказов:
  - добавление товаров в корзину из карточки товара;
  - изменение количества и удаление позиций в корзине;
  - оформление заказа и сохранение данных в таблицах заказов и товаров заказа;
  - отображение истории заказов в личном кабинете пользователя;
- проверка административных сценариев (пространство имён `App\Livewire\Admin`):
  - создание, редактирование и удаление категорий и товаров;
  - управление фильтрами и группами фильтров;
  - просмотр и изменение статусов заказов;
  - управление пользователями.

Отдельно проверялась валидация форм (регистрация, авторизация, оформление заказа, формы админ‑панели): обязательность полей, корректность формата e‑mail, телефонного номера, адреса доставки, ограничений по длине и типу данных. 5.1 Определение целевой аудитории и УТП

Для проектируемого интернет‑магазина были выделены основные сегменты целевой аудитории:
- студенты и молодые специалисты, использующие технику Apple для учёбы, работы и развлечений;
- специалисты в сфере IT, дизайна, маркетинга и медиа, для которых Mac, iPhone и iPad являются основными рабочими инструментами;
- пользователи среднего и старшего возраста, рассматривающие устройства Apple как надёжные и удобные средства связи и доступа к информации.

Уникальное торговое предложение (УТП) интернет‑магазина:
- продажа оригинальной техники Apple и качественных аксессуаров;

- прозрачные условия гарантии и сервисного обслуживания;
- несколько вариантов доставки и оплаты (включая рассрочку).

5.2 Подбор ключевых запросов и семантическое ядро

Для продвижения интернет‑магазина было сформировано семантическое ядро — список ключевых запросов, по которым пользователи ищут технику Apple и сопутствующие товары. Ядро включает:

На основе семантического ядра были определены основные разделы каталога (iPhone, iPad, Mac, Apple Watch, аксессуары) и страницы с информацией о доставке, оплате и сервисе, что позволяет распределять ключевые запросы по релевантным посадочным страницам.

А.2. Модель Product с отношениями и скоупами
php
<?php
// app/Models/Product.php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\SoftDeletes;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Database\Eloquent\Relations\BelongsToMany;
use Illuminate\Database\Eloquent\Builder;

class Product extends Model
{
    use SoftDeletes;

    protected $fillable = [
        'category_id', 'name', 'slug', 'description',
        'price', 'stock_quantity', 'sku', 'specifications',
        'image_main', 'image_gallery', 'is_active', 'is_featured'
    ];

    protected $casts = [
        'specifications' => 'array',
        'image_gallery' => 'array',
        'price' => 'decimal:2',
        'is_active' => 'boolean',
        'is_featured' => 'boolean',
    ];

    // Отношение к категории
    public function category(): BelongsTo
    {
        return $this->belongsTo(Category::class);
    }

    // Отношение к заказам (многие-ко-многим через промежуточную таблицу)
    public function orders(): BelongsToMany
    {
        return $this->belongsToMany(Order::class, 'order_product')
                    ->withPivot('quantity', 'unit_price')
                    ->withTimestamps();
    }

    // Локальный scope для активных товаров
    public function scopeActive(Builder $query): Builder
    {
        return $query->where('is_active', true);
    }

    // Локальный scope для товаров в наличии
    public function scopeInStock(Builder $query): Builder
    {
        return $query->where('stock_quantity', '>', 0);
    }

    // Аксессор для вычисляемого атрибута
    public function getFormattedPriceAttribute(): string
    {
        return number_format($this->price, 2, ',', ' ') . ' руб.';
    }
}
